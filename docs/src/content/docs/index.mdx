## What is Reagent?

Reagent is four things:

<CardGrid stagger>
  <Card title="MCP Server" icon="setting">
    `reagent serve` sits between Claude Code and every tool your AI agent can reach. Every tool
    call — whether native or proxied — flows through a 12-layer governance middleware chain:
    policy enforcement, tier classification, blocked path enforcement, secret redaction, and
    hash-chained audit logging.
  </Card>
  <Card title="Config Scaffolder" icon="puzzle">
    `reagent init` writes `.mcp.json` and installs safety hooks, behavioral policies, quality
    gates, agent teams, and developer tooling into any project in one command.
  </Card>
  <Card title="Stack Analyzer" icon="magnifier">
    `reagent catalyze` detects your project's tech stack and generates a gap analysis report
    showing which hooks, gates, and agents are missing.
  </Card>
  <Card title="Project Management" icon="list-format">
    Lightweight task tracking with JSONL event store, native MCP tools, GitHub issue sync, and a
    product-owner agent for task planning.
  </Card>
</CardGrid>

## Why Reagent?

AI coding assistants are powerful but unconstrained. Reagent adds the missing governance layer:

- **Policy enforcement** — graduated autonomy levels (L0–L3) control which tiers of tools an agent can invoke
- **Kill switch** — `reagent freeze` immediately blocks all tool calls and hook-guarded commands
- **Secret redaction** — tool arguments and outputs are scanned for AWS keys, GitHub tokens, API keys, PEM private keys, and more
- **Audit trail** — every tool invocation is logged as hash-chained JSONL with serialized writes for chain integrity
- **Multi-credential management** — switch between Claude billing accounts with Keychain-backed token storage and one-command switching
- **26 safety hooks** — settings protection, secret scanning, dangerous command interception, blocked path enforcement, Obsidian vault sync, and more
- **Quality gates** — commit and push review gates with triage scoring and agent-spawned code review
