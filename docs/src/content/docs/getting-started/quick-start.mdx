## Install and initialize

```bash
npm install -g @bookedsolid/reagent
cd your-project
reagent init
```

`reagent init` writes `.mcp.json` to your project root:

```json
{
  "mcpServers": {
    "reagent": {
      "type": "stdio",
      "command": "npx",
      "args": ["reagent", "serve"]
    }
  }
}
```

It also installs `.reagent/policy.yaml`, `.reagent/gateway.yaml`, Claude Code hooks, quality
gates, Cursor rules, agent team files, and slash commands.

**Restart Claude Code.** It picks up `.mcp.json` and spawns `reagent serve` automatically via
stdio transport. reagent loads your policy and gateway config, connects to any downstream
servers, and starts listening.

## Verify the installation

```bash
reagent check
```

## Choose a profile

```bash
# Default: client-engagement
reagent init

# BST internal projects
reagent init --profile bst-internal

# Tech stack overlays (compose on top of a base profile)
reagent init --profile nextjs
reagent init --profile lit-wc
reagent init --profile astro
reagent init --profile drupal

# Preview without writing files
reagent init --dry-run
```

See [Profiles](/reference/profiles/) for details on what each profile installs.

## Add optional integrations

```bash
# Scaffold GitHub labels, milestones, and topics
reagent init --github

# Configure Discord notifications
reagent init --discord \
  --guild-id 1234567890 \
  --alerts-channel 111 \
  --tasks-channel 222 \
  --releases-channel 333 \
  --dev-channel 444
```

## Connect additional MCP servers

Instead of listing downstream servers directly in `.mcp.json` (which would bypass reagent's
middleware), add them to `.reagent/gateway.yaml`:

```yaml
version: '1'

servers:
  filesystem:
    command: npx
    args: ['@modelcontextprotocol/server-filesystem', '/path/to/project']

  github:
    command: npx
    args: ['@modelcontextprotocol/server-github']
    env:
      GITHUB_TOKEN: '${GITHUB_PERSONAL_ACCESS_TOKEN}'
```

Restart Claude Code after editing `gateway.yaml`. reagent reconnects to all configured servers
on startup.

## Analyze your stack

```bash
# Analyze and generate a gap report
reagent catalyze

# Audit drift against the last plan
reagent catalyze --audit

# Preview without writing files
reagent catalyze --dry-run
```

This generates `catalyze-report.md` and `catalyze-report.html` listing gaps by severity.
