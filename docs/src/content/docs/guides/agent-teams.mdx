Reagent installs 89 AI agent definitions into `.claude/agents/` covering engineering, AI
platforms, and project management roles. Each agent has:

- A domain-specific system prompt and tool access list
- Zero-trust protocol (read before writing, verify before claiming, HALT compliance)
- Persona metadata (name, inspiration) grounded in real domain pioneers

## How agents work

Agents are Markdown files with frontmatter metadata. When invoked via `subagent_type` in
Claude Code's Agent tool, the agent's system prompt is loaded and the specialist operates
within its defined scope.

```
Agent tool → subagent_type: "security-engineer" → loads .claude/agents/security-engineer.md
```

All agents follow the zero-trust protocol:

- Read files before editing
- Verify state before claiming status
- Check `.reagent/HALT` before taking action
- Respect `blocked_paths` and `autonomy_level` from `policy.yaml`

## The orchestrator

The `reagent-orchestrator` is the primary routing layer. CLAUDE.md instructs the main
conversation to delegate non-trivial tasks here first:

1. Reads `.reagent/policy.yaml` and checks HALT before any work
2. Selects the right specialist agents based on the task
3. Enforces engineering processes and quality gates
4. Launches multiple specialists in parallel for throughput
5. Coordinates multi-step work across domains

**Fallback:** If the task is narrowly scoped to a single domain, the main conversation
may route directly to a specialist by using its `subagent_type` name.

## Invoking agents

Agents are invoked through Claude Code's Agent tool with the `subagent_type` parameter:

```
Use the Agent tool with subagent_type: "security-engineer" to audit the authentication flow.
```

The slash commands also invoke agents:

| Command       | Agent invoked            |
| ------------- | ------------------------ |
| `/rea`        | `reagent-orchestrator` + `ai-rea` |
| `/plan-work`  | `product-owner`          |
| `/review-pr`  | `code-reviewer` + `pr-voice-reviewer` |
| `/pm-status`  | `reagent-orchestrator`   |

For the full roster with descriptions, see [AGENTS.md](https://github.com/bookedsolidtech/reagent/blob/staging/AGENTS.md) in the repository.
