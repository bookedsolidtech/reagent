## Install and initialize

```bash
npm install -g @bookedsolid/reagent
cd your-project
reagent init
```

`reagent init` does four things:

1. Installs `@bookedsolid/reagent` as a devDependency (detects your package manager automatically)
2. Writes `.mcp.json` to your project root so Claude Code spawns `reagent serve` on session start
3. Scaffolds `.reagent/policy.yaml`, `.reagent/gateway.yaml`, Claude Code hooks, quality gates, Cursor rules, agent team files, and slash commands
4. Sets up husky git hooks for commit message validation, pre-commit checks, and pre-push gates

The `.mcp.json` it writes uses a direct `node` path for cross-package-manager compatibility:

```json
{
  "mcpServers": {
    "reagent": {
      "type": "stdio",
      "command": "node",
      "args": ["node_modules/@bookedsolid/reagent/dist/cli/index.js", "serve"]
    }
  }
}
```

:::note
If reagent is not yet installed as a local dependency (e.g., running via `npx` for the first time), init falls back to `npx @bookedsolid/reagent serve`. Run `reagent upgrade` after installing the package to migrate to the direct node path.
:::

**Restart Claude Code.** It picks up `.mcp.json` and spawns `reagent serve` automatically via
stdio transport. reagent loads your policy and gateway config, connects to any downstream
servers, and starts listening.

## Verify the installation

```bash
reagent check
```

## Choose a profile

```bash
# Default: client-engagement
reagent init

# BST internal projects
reagent init --profile bst-internal

# Tech stack overlays (compose on top of a base profile)
reagent init --profile nextjs
reagent init --profile lit-wc
reagent init --profile astro
reagent init --profile drupal

# Preview without writing files
reagent init --dry-run
```

See [Profiles](/reference/profiles/) for details on what each profile installs.

## Add optional integrations

```bash
# Scaffold GitHub labels, milestones, and topics
reagent init --github

# Configure Discord notifications
reagent init --discord \
  --guild-id 1234567890 \
  --alerts-channel 111 \
  --tasks-channel 222 \
  --releases-channel 333 \
  --dev-channel 444
```

## Connect additional MCP servers

Instead of listing downstream servers directly in `.mcp.json` (which would bypass reagent's
middleware), add them to `.reagent/gateway.yaml`:

```yaml
version: '1'

servers:
  filesystem:
    command: npx
    args: ['@modelcontextprotocol/server-filesystem', '/path/to/project']

  github:
    command: npx
    args: ['@modelcontextprotocol/server-github']
    env:
      GITHUB_TOKEN: '${GITHUB_PERSONAL_ACCESS_TOKEN}'
```

Restart Claude Code after editing `gateway.yaml`. reagent reconnects to all configured servers
on startup.

## Analyze your stack

```bash
# Analyze and generate a gap report
reagent catalyze

# Audit drift against the last plan
reagent catalyze --audit

# Preview without writing files
reagent catalyze --dry-run
```

This generates `catalyze-report.md` and `catalyze-report.html` listing gaps by severity.

## Keeping reagent up to date

After upgrading the npm package, run `upgrade` to re-sync hooks and migrate configs:

```bash
npm update -g @bookedsolid/reagent   # or: npm update -D @bookedsolid/reagent
reagent upgrade
```

`reagent upgrade` re-syncs husky hooks, merges new policy fields into `policy.yaml`, ensures
reagent is a devDependency, and migrates `.mcp.json` from legacy patterns. See [CLI Commands](/reference/cli-commands/) for all flags.
