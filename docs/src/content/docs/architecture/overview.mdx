## Key design decisions

### Onion middleware

The MCP server uses Koa-style onion middleware where each layer wraps the next. This means audit (outermost) captures everything, including denials from inner layers.

### Append-only event store

The task management layer uses an append-only JSONL event log rather than mutable records. Current state is materialized by replaying events. This ensures no data loss and safe concurrent writes.

### Convention-based classification

Unknown tools are classified by prefix pattern rather than requiring explicit configuration. This lets the gateway handle new downstream servers sensibly without configuration changes.

### Hash-chained audit

Audit records form a hash chain where each record includes the hash of the previous record. This provides tamper evidence — modifying any record invalidates the chain from that point forward.
