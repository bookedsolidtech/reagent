Reagent installs 26 Claude Code hooks that enforce security, quality, project management, and
integration policies. Hooks are shell scripts that run as PreToolUse or PostToolUse interceptors.

Every hook checks for `.reagent/HALT` at the top of its script. If the file exists, the hook
returns exit code 2 (hard block) immediately.

## Hook architecture

Hooks use a shared library (`hooks/_lib/common.sh`) providing:

| Function       | Purpose                                               |
| -------------- | ----------------------------------------------------- |
| `reagent_root` | Find the `.reagent/` directory by walking up from cwd |
| `check_halt`   | Exit with code 2 if `.reagent/HALT` exists            |
| `require_jq`   | Verify jq is available                                |
| `json_output`  | Build structured JSON response (block/allow/advisory) |
| `triage_score` | Score a diff as trivial/standard/significant          |

## Exit code convention

| Code | Meaning             |
| ---- | ------------------- |
| `0`  | Allow the tool call |
| `2`  | Block the tool call |

## Security hooks (11)

### `settings-protection.sh` (PreToolUse: Write, Edit)

**P0 Critical.** Prevents agents from modifying their own safety rails. Blocks writes to `.claude/settings.json`, `.claude/hooks/*`, `.husky/*`, `.reagent/policy.yaml`, `.reagent/HALT`, and `.reagent/review-cache.json`. Includes case-insensitive bypass detection and URL-decode normalization.

### `blocked-paths-enforcer.sh` (PreToolUse: Write, Edit)

**P0 Critical.** Reads `blocked_paths` from `.reagent/policy.yaml` and blocks writes to matching paths. Supports directory prefix matching, glob patterns, and exact match.

### `dangerous-bash-interceptor.sh` (PreToolUse: Bash)

Intercepts dangerous shell commands:

| ID  | Blocked Pattern                                      |
| --- | ---------------------------------------------------- |
| H1  | `git push --force` / `-f` to protected branches      |
| H2  | `rm -rf /` or `rm -rf ~`                             |
| H3  | `chmod 777`                                          |
| H4  | `curl \| sh`, `wget \| sh` (pipe-to-shell)           |
| H5  | `> /dev/sda` (disk overwrite)                        |
| H6  | Fork bomb                                            |
| H7  | `mkfs` (disk format)                                 |
| H8  | `dd if=` (disk copy)                                 |
| H9  | `.env` file access via cat/less/head/tail            |
| H10 | `shutdown`, `reboot`, `halt`, `poweroff`             |
| H11 | `kill -9`, `killall`, `pkill`                        |
| H12 | `iptables` / `ufw` (firewall modification)           |
| H13 | `git push --no-verify`                               |
| H14 | `git -c core.hooksPath=` (hook path override)        |
| H15 | `REAGENT_BYPASS` environment variable                |
| H16 | Alias/function definitions containing bypass strings |

### `secret-scanner.sh` (PreToolUse: Write, Edit)

Scans file content for secrets: AWS Access Keys, PEM private key headers, GitHub PATs, Stripe live keys, and generic `SECRET=`/`PASSWORD=`/`TOKEN=` assignments. Allows placeholders, `process.env` references, and `.env.example` files.

### `env-file-protection.sh` (PreToolUse: Write, Edit)

Blocks writes to `.env` files. Allows `.env.example` and `.env.template`.

### `attribution-advisory.sh` (PreToolUse: Bash)

When `block_ai_attribution` is enabled in `policy.yaml`, blocks `gh pr create`, `gh pr edit`, and `git commit` commands containing AI attribution patterns (`Co-Authored-By` with AI names, `Generated with [Tool]` footers, etc.).

### `symlink-guard.sh` (PreToolUse: Write, Edit)

Detects symlink traversal attempts — blocks writes to paths that resolve through a symlink outside the project root.

### `network-exfil-guard.sh` (PreToolUse: Bash)

Blocks `curl`/`wget`/`fetch` commands targeting non-allowlisted external hosts.

### `import-guard.sh` (PreToolUse: Write, Edit)

Flags dangerous import patterns: `eval`, `Function()`, dynamic `require()` with user-controlled input.

### `git-config-guard.sh` (PreToolUse: Bash)

Blocks `git config` commands that override hook paths, rewrite signing, or modify credential helpers.

### `ci-config-protection.sh` (PreToolUse: Write, Edit)

Blocks writes to `.github/workflows/` and other CI configuration files.

## Quality gate hooks (4)

### `commit-review-gate.sh` (PreToolUse: Bash)

Intercepts `git commit` and applies triage-based review:

| Level           | Criteria                                        | Action                                         |
| --------------- | ----------------------------------------------- | ---------------------------------------------- |
| **Trivial**     | Fewer than 20 changed lines, no sensitive paths | Pass immediately                               |
| **Standard**    | 20-200 changed lines                            | Check review cache; pass if cached             |
| **Significant** | More than 200 lines OR sensitive paths          | Block; instruct agent to spawn `code-reviewer` |

### `push-review-gate.sh` (PreToolUse: Bash)

Intercepts `git push`. Analyzes the full diff against the target branch and checks review cache. On cache miss for standard/significant changes, blocks with instructions to spawn `code-reviewer` and `security-engineer` agents.

### `architecture-review-gate.sh` (PostToolUse: Write, Edit)

**Advisory only (never blocks).** Flags writes to architecture-sensitive paths (`src/types/`, `src/gateway/`, `src/config/`, `package.json`, etc.).

### `changeset-security-gate.sh` (PreToolUse: Write, Edit)

Validates changeset files before they are written. Prevents injection of malicious content into the changeset workflow.

## Project management hooks (3)

### `task-link-gate.sh` (PreToolUse: Bash)

**Opt-in** (requires `task_link_gate: true` in policy.yaml). Checks that commit messages contain a task ID reference (`T-NNN` format). Allows merge commits, version bumps, and `chore:`/`style:`/`ci:` prefixed commits.

### `dependency-audit-gate.sh` (PreToolUse: Bash)

Intercepts `npm install`, `pnpm add`, `yarn add`, and `npx` commands. Verifies each package exists in the npm registry before allowing the install.

### `pr-issue-link-gate.sh` (PreToolUse: Bash)

Requires every PR created via `gh pr create` to reference a GitHub issue in its body or title.

## Safety hooks (4)

### `output-validation.sh` (PostToolUse)

Scans tool output returned to the AI for secrets before they enter the model's context.

### `file-size-guard.sh` (PreToolUse: Write)

Blocks writes of files exceeding a configurable size threshold.

### `rate-limit-guard.sh` (PreToolUse)

Tracks tool call frequency per session and blocks when a per-minute threshold is exceeded.

### `security-disclosure-gate.sh` (PreToolUse: Bash)

Intercepts `gh issue create` and similar commands. Blocks public disclosure of security vulnerabilities; routes to the configured disclosure mode (issues, security advisories, etc.).

## Integration hooks (4)

### `reagent-notify.sh` (PostToolUse: Bash)

Sends Discord notifications on significant events (commits, PR creation, pushes). Requires `discord_ops` configured in `gateway.yaml` and `DISCORD_BOT_TOKEN` in the environment.

### `reagent-obsidian-journal.sh` (PostToolUse: Stop)

Fires when a Claude Code session ends. Appends a session summary to the daily note in your Obsidian vault via `obsidian daily:append`. Includes project name, session identifier, timestamp, and task state counts (completed, in progress, blocked, backlog) from `.reagent/tasks.jsonl`.

Requires: `sync.journal: true` in `gateway.yaml` Obsidian config. See [Obsidian Vault Integration](/guides/obsidian-vault/).

### `reagent-obsidian-precompact.sh` (PreToolUse: Compact)

Fires before Claude Code compacts context. Creates a session knowledge note in the vault's sessions directory with frontmatter tagging the project, date, session ID, and extraction engine.

Requires: `sync.precompact: true` in `gateway.yaml` Obsidian config (disabled by default).

### `reagent-obsidian-tasks.sh` (PostToolUse: task_create, task_update)

Fires after `task_create` and `task_update` MCP tool calls. Materializes each task as an individual Obsidian note with typed frontmatter (task ID, project, status, urgency, assignee). Sync is one-way: JSONL to Obsidian.

Requires: `sync.tasks: true` in `gateway.yaml` Obsidian config. See [Obsidian Vault Integration](/guides/obsidian-vault/).
