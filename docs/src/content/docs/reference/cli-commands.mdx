## Commands

| Command                         | Description                                                        |
| ------------------------------- | ------------------------------------------------------------------ |
| `reagent init`                  | Install reagent config into the current directory                  |
| `reagent serve`                 | Start the MCP server on stdio (spawned by Claude Code via .mcp.json) |
| `reagent check`                 | Verify which reagent components are installed                      |
| `reagent freeze --reason "..."` | Create `.reagent/HALT` — suspends all tool calls and hook actions  |
| `reagent unfreeze`              | Remove `.reagent/HALT` — resumes tool calls and hook actions       |
| `reagent catalyze`              | Analyze project stack and generate gap report                      |
| `reagent upgrade`               | Re-sync hooks, merge policy, migrate .mcp.json, install devDep    |
| `reagent cache check <sha>`     | Check review cache for a file SHA                                  |
| `reagent cache set <sha> <res>` | Store a review result (pass/fail/advisory)                         |
| `reagent cache clear`           | Clear all cached review results                                    |
| `reagent account <subcommand>`  | Multi-credential management (add, list, switch, rotate, remove)    |
| `reagent obsidian <subcommand>` | Obsidian vault integration commands                                |
| `reagent help`                  | Show usage help                                                     |

## `reagent init`

See [Config Scaffolder](/guides/config-scaffolder/) for full details.

`reagent init` performs these steps in order:

1. Detects your package manager (npm, yarn, pnpm, bun) and installs `@bookedsolid/reagent` as a devDependency
2. Writes `.mcp.json` with a direct `node` path for cross-PM compatibility
3. Scaffolds `.reagent/policy.yaml` and `.reagent/gateway.yaml`
4. Installs Claude Code hooks (26 scripts) into `.claude/hooks/`
5. Installs agent team definitions (89 agents) into `.claude/agents/`
6. Installs slash commands into `.claude/commands/`
7. Writes Cursor rules, husky git hooks, and `CLAUDE.md`

| Flag                      | Description                                    | Default             |
| ------------------------- | ---------------------------------------------- | ------------------- |
| `--profile <name>`        | Profile to install                             | `client-engagement` |
| `--base-profile <name>`   | Base profile to use with a tech stack profile  | `client-engagement` |
| `--dry-run`               | Preview what would be installed without writes | —                   |
| `--github`                | Scaffold GitHub labels, milestones, and topics | —                   |
| `--discord`               | Configure Discord notifications in gateway     | —                   |
| `--guild-id <id>`         | Discord server ID (used with `--discord`)      | —                   |
| `--alerts-channel <id>`   | Discord channel for security alerts            | —                   |
| `--tasks-channel <id>`    | Discord channel for task events                | —                   |
| `--releases-channel <id>` | Discord channel for release events             | —                   |
| `--dev-channel <id>`      | Discord channel for dev activity               | —                   |
| `--obsidian`              | Enable Obsidian vault integration              | —                   |
| `--vault-path <path>`     | Absolute path to the Obsidian vault directory  | —                   |

## `reagent serve`

Starts the MCP server on stdio. This command is called by Claude Code automatically via the
`.mcp.json` declaration written by `reagent init`. You do not normally invoke this manually.

reagent loads `.reagent/policy.yaml` and `.reagent/gateway.yaml` from the current directory,
connects to any configured downstream servers, and starts listening on stdio.

## `reagent freeze`

Creates `.reagent/HALT` with the provided reason as the file content. Every hook checks for
this file at the top of its script. Every tool call checks it at the kill-switch middleware
layer. Nothing gets through while the file exists.

| Flag              | Description                        | Default         |
| ----------------- | ---------------------------------- | --------------- |
| `--reason <text>` | Reason for freeze (stored in HALT) | `Manual freeze` |

## `reagent unfreeze`

Removes `.reagent/HALT`. Agent operations resume immediately.

## `reagent upgrade`

Re-syncs installed hooks, merges new policy fields into `policy.yaml`, ensures reagent is
installed as a devDependency, and migrates `.mcp.json` from the legacy `npx` pattern to the
direct `node` path (for pnpm compatibility).

Run this after upgrading the `@bookedsolid/reagent` package:

```bash
npm update -g @bookedsolid/reagent
reagent upgrade
```

**Upgrade steps performed:**

1. **Husky hooks** — re-syncs `commit-msg`, `pre-commit`, `pre-push` from the package (only updates hooks that already exist in `.husky/`)
2. **Policy merge** — YAML-aware merge that adds missing canonical sections (e.g., `context_protection`), updates the `installed_by` version stamp, and preserves existing values and YAML comments
3. **Package dependency** — ensures `@bookedsolid/reagent` is a devDependency with `^{currentVersion}` range (detects PM and workspace roots)
4. **`.mcp.json` migration** — detects the broken `npx reagent serve` pattern and migrates to `node node_modules/@bookedsolid/reagent/dist/cli/index.js serve`

| Flag                    | Description                                                                     |
| ----------------------- | ------------------------------------------------------------------------------- |
| `--dry-run`             | Print what would be updated without writing                                     |
| `--clean-blocked-paths` | Replace blanket `.reagent/` blocked path with granular entries for policy/audit  |

The `--clean-blocked-paths` flag replaces a blanket `.reagent/` entry in `blocked_paths` with
granular entries that protect security-critical files (policy, audit, HALT) while allowing
agents to write to operational files like `tasks.jsonl` and `review-cache.json`.

## `reagent cache`

The review cache stores code review results to avoid redundant agent reviews on unchanged code.

```bash
# Check if a file has a cached review
reagent cache check abc123 --branch main --base def456

# Store a review result
reagent cache set abc123 pass --branch main --reviewer code-reviewer --findings 0 --ttl 86400

# Clear all cache entries
reagent cache clear
```

| Flag                | Description               | Default   |
| ------------------- | ------------------------- | --------- |
| `--branch <name>`   | Branch name for cache key | `""`      |
| `--base <commit>`   | Base commit for cache key | `""`      |
| `--reviewer <name>` | Reviewer agent name       | `unknown` |
| `--findings <n>`    | Number of findings        | `0`       |
| `--ttl <seconds>`   | Cache entry TTL           | `86400`   |

Cache file: `.reagent/review-cache.json`, keyed on `${branch}:${baseCommit}:${fileSHA256}`.

## `reagent obsidian`

Obsidian vault integration commands. See [Obsidian Vault Integration](/guides/obsidian-vault/) for full details.

| Subcommand                                    | Description                                              |
| --------------------------------------------- | -------------------------------------------------------- |
| `reagent obsidian sync [--target <t>]`        | Sync enabled targets to the vault                        |
| `reagent obsidian sync --dry-run`             | Preview what would be written                            |
| `reagent obsidian status`                     | Show Obsidian config, CLI availability, enabled targets  |
| `reagent obsidian health`                     | Show vault health metrics (orphans, unresolved links)    |
| `reagent obsidian journal`                    | Manually trigger a session journal entry in daily note   |

Valid `--target` values: `kanban`, `context`, `wiki`, `tasks`.

## `reagent account`

Multi-credential management for switching between Claude billing accounts. See
[Multi-Credential Management](/guides/multi-credential-management/) for the full guide and
[Account Security Model](/architecture/account-security-model/) for threat model details.

| Subcommand | Description |
|---|---|
| `reagent account add <name>` | Register a new account via OAuth login, store in Keychain |
| `reagent account list` | Show all registered accounts and active status |
| `reagent account env <name>` | Output shell export commands for `eval` |
| `reagent account env --clear` | Output shell unset commands to clear override |
| `reagent account check [--all]` | Validate token health (expiry, keychain access) |
| `reagent account verify [--all]` | Verify account identity via Anthropic API (email, plan, billing) |
| `reagent account whoami` | Show active account details |
| `reagent account rotate <name>` | Re-authenticate and store new token |
| `reagent account remove <name>` | Delete keychain entry and remove from config |
| `reagent account setup-shell` | Print shell function and completions for `rswitch` |

| Flag | Description | Default |
|---|---|---|
| `--shell <zsh\|bash\|fish>` | Target shell for `setup-shell` | `zsh` |
| `--all` | Apply to all accounts (for `check` and `verify` subcommands) | Active only |

## Slash commands

reagent installs these slash commands into `.claude/commands/` for use within Claude Code conversations:

| Command       | Description                                                                |
| ------------- | -------------------------------------------------------------------------- |
| `/tasks`      | Render a markdown table of current tasks from `.reagent/tasks.jsonl`       |
| `/plan-work`  | Invoke the product-owner agent to propose and create tasks for a goal      |
| `/restart`    | Session handoff — save state on spin-down, orient from saved state on spin-up |
| `/rea`        | Invoke the REA (Reactive Execution Agent) for autonomous team orchestration |
| `/pm-status`  | Scan open PRs, report pipeline state, merge ready work, staging promotion  |
| `/review-pr`  | Fetch a PR diff, run code-reviewer analysis, post findings in owner voice  |
